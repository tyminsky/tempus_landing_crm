# Tempus CRM Integration

Проєкт посадкової сторінки (LP) під рекламу в Google для тесту гіпотези:
знайти клієнтів, яким треба інтегрувати інтернет-магазин з CRM, налаштувати
аналітику та якісну IP-телефонію — разом або окремо.

> Назва в корені — `Tempus CRM Integration`. Tempus OS синхронізує проєкт за
> ID-шником, а не за назвою (тех. примітка з брифінгу).

## Місія
Побудувати найкращий сервіс у Європі для малого бізнесу та середніх
інтернет-магазинів: швидко, якісно, одразу працює, є підтримка, навчання,
відео та відповідь на будь-яке питання. Сучасна AI-компанія — весь проєкт
ведемо через AI.

## Пріоритети (з брифінгу Serg)
1. **Швидкість** — найважливіше.
2. **Якість** — середня/адекватна; передусім **якість тексту**.
3. **Дизайн** — значення не має (це тест гіпотези).

## Структура проєкту
```
Tempus_LP_CRM/
├─ README.md          ← цей файл
├─ BRIEF.md           ← структурований бриф (головний документ стратегії)
├─ OFFERS.md          ← офери та тексти лендінга (чернетка)
├─ HEADLINES.md       ← фінальна добірка заголовків (після конвеєра агентів)
├─ PLAN.md            ← план розробки лендінга (етапи, рішення)
├─ research/          ← звіти агентів-дослідників (Sonnet)
│  ├─ icp-analysis.md      ← болі сегментів словами клієнтів
│  ├─ semantics.md         ← пошукові кластери Google Ads, message match
│  ├─ competitors.md       ← розвідка конкурентів, вільні позиції
│  ├─ headlines-draft.md   ← чернетка копірайтера (25 H1, RSA)
│  └─ critique.md          ← протокол критики (вердикти по кожному H1)
├─ transcripts/       ← транскрипти зустрічей, думки, чати
│  └─ 2026-06-12-serg-briefing.md
└─ ads/               ← верстка (база Tempus + наша сторінка)
   ├─ seo-ads.html                ← базова верстка (НЕ редагувати — гайдлайн)
   ├─ crm-integration.html       ← НАШ лендінг (копія бази + наші тексти)
   ├─ audit_crm_integration.css  ← нові стилі (тарифи, мем-тумблер)
   ├─ audit_crm_integration.js   ← нові скрипти (логіка тумблера)
   ├─ MARKUP_GUIDELINES.md       ← правила верстки (BEM, секції)
   └─ css/ js/ img/ fonts/       ← ресурси бази
```

> `SEO_ADS_BUILD_GUIDELINES.md` (корінь) — правила розширення бази без шкоди.

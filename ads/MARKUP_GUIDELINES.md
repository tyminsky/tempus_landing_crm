# Правила верстки — чернетка

Документ описує тільки верстку: іменування класів та організацію CSS-файлів. Правила не прив'язані до конкретного препроцесора — формулювання працюють і для чистого CSS, і для SASS/SCSS/Less.

---

## 1. Іменування класів

### 1.1 Методологія — BEM

Усі класи будуються за схемою **Block — Element — Modifier**:

- **Блок** — самостійна сутність: `.shortcard`, `.header`, `.breadcrumbs`.
- **Елемент** — частина блоку, не існує поза ним; відділяється двома підкресленнями: `.shortcard__title`, `.header__row`, `.form_field__input`.
- **Модифікатор** — варіант блоку або елемента; відділяється двома дефісами: `.btn-submit--white`, `.form_field--50`, `.edit_btn--bordered`.

### 1.2 Регістр

- Імена блоків і елементів — **snake_case** (нижній регістр, слова через `_`): `home_map`, `contacts_page`, `info_card`.
- Усередині елемента можна використовувати дефіс для уточнення складеного імені: `home_map__contact-itm`, `home_map__map-dot`.
- Не використовуйте camelCase і UPPERCASE у класах.

### 1.3 Префікси за призначенням

Прийняті в проєкті групи:

- Сторінки — суфікс `_page`: `.contacts_page`, `.home_page`, `.catalog_page` (виставляється на `<body>`).
- Кнопки — `.btn-*` або суфікс `_btn`: `.btn-submit`, `.read_more_btn`, `.login_btn`, `.next_step_btn`, `.edit_btn`.
- Іконки в розмітці — `.ico`, `.svg-ico`.
- Форми — `.form_field`, `.form_field__label`, `.form_field__input`, `.form_consent`, `.form_dropdown`.

### 1.4 Стани (utility-класи)

Стани — короткі однослівні класи без префіксу блока, які додаються/знімаються JS:

- `.active` — активний таб/елемент.
- `.sticky` — закріплений хедер.
- `.disabled` — задизейблений елемент.
- `.hidden` — повністю прихований (`display: none !important`).
- `.no-scrollbar`, `.no-animate`, `.menu-is-open` — глобальні стани, виставляються на `<body>`.

Стилі під стани пишемо як комбінацію: `.shortcard__tag.top`, `.header.sticky`, `.form_dropdown.active`.

### 1.5 Що не можна

- Не плодити синоніми: для однієї суті — один клас. Якщо вже є `.shortcard__title`, не додавайте паралельно `.shortcardTitle` або `.title-shortcard`.
- Не вкладати модифікатори всередину елементів через подвійне `--`: правильно `.shortcard__buy_btn` + `.shortcard__buy_btn--small`, а не `.shortcard__buy_btn__small`.
- Не використовувати теги як селектори глобально (`button { ... }`). Дозволено лише у вузькому контексті блоку: `.breadcrumbs ul { ... }`.
- Не використовувати `id` для стилізації — лише класи.
- Не використовувати `!important`, крім явних утиліт (`.hidden`).
- Не використовувати класи без вкладеності в блок на 1му рівні. `.title` не можна додавати, тільки `.shortcard .title`

## 2. HTML-розмітка

### 2.1 Секції сторінки

Кожен самостійний блок-секція на сторінці — це **`<section>`** з власним класом-блоком (BEM):

```html
<section class="home_promo">
  <div class="container">
    <!-- контент секції -->
  </div>
</section>
```

- `<div>` для секцій не використовуємо — лише `<section>`.
- Виняток — суто технічні обгортки (`#page-wrap`, оверлеї попапів) і дрібні всередині-секційні блоки.

### 2.2 Обмежувач ширини

Усередині кожної секції перший дочірній елемент — **`<div class="container">`**, який обмежує ширину контенту і дає бічні відступи через `--container-padding`. Увесь корисний контент секції лежить уже всередині нього.

```html
<section class="contacts_page__cta">
  <div class="container">
    <h2 class="contacts_page__cta-title">Contáctanos</h2>
    <!-- ... -->
  </div>
</section>
```

Якщо секції потрібен повношириний фон/декор — фон вішаємо на саму `<section>`, а контент усе одно тримаємо в `.container`.

### 2.3 Заголовки

Теги `<h1>`–`<h6>` **не використовуємо як селектори** в CSS. Усі стилі — лише через клас:

```html
<h1 class="page_title">Наші контакти</h1>
<h2 class="home_about__title">Про нас</h2>
<div class="home_about__suptitle">...</div>
```

Це потрібно, щоб у будь-який момент тег можна було замінити (наприклад `<h2>` → `<div>`) без зміни стилів — корисно і для SEO-перестановок, і для рефакторингу. Семантичний рівень заголовка визначає верстальник за структурою сторінки, але візуальний стиль завжди йде через клас.

Заборонено:

```css
/* ❌ ні */
.home_about h2 { font-size: 32px; }
h1 { color: $accent; }
```

Дозволено:

```css
/* ✅ так */
.home_about__title { font-size: 32px; }
.page_title { color: $accent; }
```

### 2.4 Загальні правила розмітки

- Один `<h1>` на сторінку — головний заголовок (зазвичай `<h1 class="page_title">`).
- Кнопки дії — `<button>`, переходи — `<a href="...">`. Не міняти місцями.
- Списки, breadcrumbs, навігація, таби — `<ul><li>`.
- Форми: `<label>` + `<input>` / `<select>` / `<textarea>`, обгортка — `.form_field`, обов'язкові поля — додатковий клас `.required` на обгортці.
- `<img>` завжди має `alt`, `width`, `height` (знижує CLS).
- `target="_blank"` — лише з `rel="noopener"`.
- Без інлайн `style="..."`, крім динамічних значень (наприклад `style="width: 80%;"` для прогрес-бара).
- Стани відображення — через клас `.hidden`, а не `style="display: none"`.

## 3. Структура CSS-файлів

Правила сформульовані для будь-якого препроцесора. Якщо проєкт на чистому CSS — групуйте через `@import` або інструмент збірки (PostCSS, Vite); якщо на SASS/SCSS — через `@import` / `@use`.

### 3.1 Точка входу

Є **один головний файл** (`styles.css` / `styles.sass`), у який підключаються всі модулі **в чіткому порядку**. Не імпортуйте файли посеред дерева — лише з точки входу.

### 3.2 Порядок шарів (cascade layers)

```
1. vars            — змінні (кольори, брейкпоінти, шрифти, CSS custom properties)
2. mixins/helpers  — функції, міксини, %extends (для препроцесорів)
3. fonts           — @font-face декларації
4. reset           — нормалізація стилів через `reset.css` (Eric Meyer's reset або еквівалент)
5. base            — глобальні стилі тегів (body, a, input, .container, ...)
6. components      — універсальні UI-компоненти (кнопки, форми, тултіпи)
7. global          — глобальні блоки сайту (header, footer, popup, cart)
8. partials        — багаторазові партіали (картки, breadcrumbs, пагінація)
9. sections        — стилі секцій сторінок (групуються в підпапки по сторінках)
10. vendors        — стилі сторонніх бібліотек (підключаються окремо в head)
```

Порядок важливий: компоненти не повинні залежати від секцій, секції можуть перевизначати компоненти.

### 3.3 Файли і папки

- **Один компонент = один файл.** Назва файлу = назві основного блоку BEM: `_shortcard.css`, `_home_promo.css`.
- Файли-партіали — з префіксом `_` (якщо це підтримує препроцесор) або просто згруповані в папці.
- Структура папок : `global/` для глобальних блоків, `partials/` для багаторазових, `sections/<page>/` для секцій сторінок.
- Нові файли стилів **обов'язково** додавайте в головний файл у правильний шар.

### 3.4 Структура селекторів усередині файлу

Один блок = один файл = один кореневий селектор + його елементи й модифікатори:

```css
.shortcard { ... }
.shortcard__thumb { ... }
.shortcard__img { ... }
.shortcard__title { ... }
.shortcard__price { ... }
.shortcard--featured { ... }
```

Не змішуйте в одному файлі непов'язані блоки. Якщо в `_shortcard.css` починаєте писати `.wishlist_card` — створюйте `_wishlist_card.css`.

### 3.5 Брейкпоінти (mobile-first)

Базові стилі пишемо для мобільного, далі **тільки нарощуємо** через `min-width` медіа:

```
$mobile-width:      359px
$tablet-width:      768px
$laptop-width:      1024px
$pc-width:          1200px
$pc-hd-width:       1400px
$pc-full-hd-width:  1800px
```

Використовуйте лише ці брейкпоінти. Нові значення без обговорення з командою не вводимо.

Медіа-запити — поруч із селектором, а не в окремих секціях у кінці файлу:

```css
.shortcard__title {
  font-size: 11px;
}
@media (min-width: 768px) {
  .shortcard__title { font-size: 13px; }
}
```

(У препроцесорі — через міксин `+tablet` / `@include tablet`.)

### 3.6 Адаптація складних блоків

Якщо блок має багато внутрішніх елементів і їхнє розташування **змінюється між брейкпоінтами**, для адаптації використовуємо позиціонування на рівні контейнера, а не «висмикування» окремих дітей:

- **Grid-блоки** — через `grid-template-areas` + `grid-area` на дітях. Це дає змогу повністю перебудовувати макет, змінюючи лише шаблон у контейнері.

  ```css
  .home_calc {
    display: grid;
    grid-template-areas:
      "title"
      "img"
      "form";
  }
  .home_calc__title { grid-area: title; }
  .home_calc__img   { grid-area: img; }
  .home_calc__form  { grid-area: form; }

  @media (min-width: 1024px) {
    .home_calc {
      grid-template-columns: 1fr 1fr;
      grid-template-areas:
        "title img"
        "form  img";
    }
  }
  ```

- **Flex-блоки** — порядок елементів адаптуємо через `order` на дітях, а не через зміну розмітки чи дублювання блоків.

  ```css
  .product_page__row { display: flex; flex-direction: column; }
  .product_page__gallery { order: 1; }
  .product_page__info    { order: 2; }

  @media (min-width: 1024px) {
    .product_page__row { flex-direction: row; }
    .product_page__info    { order: 1; }
    .product_page__gallery { order: 2; }
  }
  ```

Це тримає HTML-розмітку незмінною в усіх макетах, а перестановки лишаються повністю в CSS.

### 3.7 Нестандартні форми (`clip-path`)

Картинки і блоки **непрямокутної форми** (зрізані кути, фігурні маски) робимо через `clip-path: polygon(...)`, **а не** SVG-маски / PNG з прозорістю / накладені фігури-обгортки.

Координати точок задаємо у форматі **`calc(... - Npx)` від кутів** — так форма коректно адаптується під будь-яку ширину/висоту блоку без перерахунку відсотків:

```css
.info_card {
  clip-path: polygon(
    0 0,
    calc(100% - 20px) 0,
    100% 20px,
    100% 100%,
    20px 100%,
    0 calc(100% - 20px)
  );
}
```

- Розмір «зрізу» (`20px`) виносимо в змінну, якщо він повторюється в кількох блоках.
- Для різних брейкпоінтів — той самий `clip-path`, але з іншим значенням зрізу:

  ```css
  .info_card { --clip-cut: 12px; }
  @media (min-width: 1024px) { .info_card { --clip-cut: 20px; } }

  .info_card {
    clip-path: polygon(
      0 0,
      calc(100% - var(--clip-cut)) 0,
      100% var(--clip-cut),
      100% 100%,
      var(--clip-cut) 100%,
      0 calc(100% - var(--clip-cut))
    );
  }
  ```

Не використовуйте **відсотки** для точок зрізу (`polygon(0 0, 95% 0, ...)`) — на різних розмірах форма поплине, бо зріз буде непропорційний.

### 3.8 Кольори, відступи, шрифт

- Усі кольори, основні відступи й сім'ї шрифтів — лише через змінні. Хардкод (`#009628`, `728px`) у компонентах заборонений.
- Динамічні значення, які можуть змінюватися runtime або залежно від медіа (наприклад `--container-padding`), оголошуйте через CSS custom properties на `:root` / `body`.

**Типографіка:**

- `letter-spacing` задаємо в **`em`**, не в `px` — щоб трекінг масштабувався разом із розміром шрифту:

  ```css
  /* ✅ так */
  .home_promo__title { letter-spacing: -0.02em; }
  /* ❌ ні */
  .home_promo__title { letter-spacing: -0.5px; }
  ```

- `line-height` за замовчуванням — **`1.2`** (unitless), якщо в дизайні не вказано інше. Не використовуйте `line-height` у `px` без потреби — unitless значення коректно успадковується і масштабується:

  ```css
  /* ✅ так */
  .shortcard__title { line-height: 1.2; }
  /* ❌ ні без обґрунтування */
  .shortcard__title { line-height: 18px; }
  ```

### 3.9 Анімації

- Використовуйте `transition` лише для **конкретних** властивостей, не `transition: all`.
- Стандартна тривалість — 0.35s, easing — `ease`. Відхиляйтеся лише за дизайн-вимогою.
- Тривалі / складні анімації (keyframes) — у тому ж файлі, що й блок, який їх використовує.

### 3.10 Порядок властивостей у селекторі

Рекомендована послідовність (зустрічається у проєкті):

1. `position`, `top` / `right` / `bottom` / `left`, `z-index`
2. `display`, flex/grid-властивості, `align-*`, `justify-*`, `gap`
3. `width`, `height`, `min/max-*`, `aspect-ratio`
4. `padding`, `margin`
5. `border`, `border-radius`, `background`
6. `font-*`, `line-height`, `color`, `text-*`, `letter-spacing`
7. `transition`, `transform`, `opacity`
8. Псевдоелементи (`:before`, `:after`), псевдокласи (`:hover`, `:focus`)
9. Модифікатори (`.block--mod`), вкладені елементи (`.block__elem`)
10. Медіа-запити

### 3.11 Псевдоелементи `::before` / `::after`

Декоративні елементи (іконки-стрілочки, риски, рамки-градієнти, бекграунд-оверлеї, кастомні чекбокси/радіо) реалізуємо через `::before` / `::after`, **а не додаткові HTML-теги**.

Так розмітка лишається семантично чистою, а декор — повністю під контролем CSS.

```css
.read_more_btn::after {
  content: '';
  width: 30px;
  height: 38px;
  border-radius: 12px;
  background: url(...) center no-repeat;
}
```

Якщо елемент несе сенс (текст, посилання, дані для скрін-рідерів) — це **не** псевдоелемент, а звичайний тег.

### 3.12 Інтерактивні стани

Для кожного інтерактивного елемента **обов'язково** прописуємо стани:

- Кнопки, посилання, кліковані картки, табі — `:hover` і `:active`.
- Поля вводу (`input`, `textarea`, `select`) — додатково `:focus` (і `:focus-visible`, де доречно).
- Задизейблений стан — через `[disabled]` або клас `.disabled`.

```css
.btn-submit { background: #000; color: #fff; }
.btn-submit:hover { background: $accent; }
.btn-submit:active { background: $green-dark; }

.form_field__input { border: 1px solid #ccc; }
.form_field__input:focus { border-color: $accent; outline: none; }
```

Не залишайте інтерактивний елемент «німим»: відсутність відгуку на наведення/натискання — баг, а не дизайн-рішення.

### 3.14 Ізоляція стилів сторінки (page scope)

Кожна нова сторінка **зобов'язана** мати унікальний CSS-клас на елементі `<body>`. Усі стилі, специфічні для цієї сторінки, будуються **виключно від цього кореневого класу**.

**Навіщо:**
- Унеможливлює конфлікти між однойменними класами на різних сторінках.
- Дозволяє безпечно перевикористовувати загальні BEM-блоки (`.hero`, `.shortcard`, `.cta`) з різними стилями на кожній сторінці — без модифікаторів і перевизначень у глобальному просторі.
- Локалізує відладку: одразу видно, до якої сторінки відноситься правило.

**Правило іменування:**
Клас сторінки відповідає угоді з п. 1.3 — суфікс `_page`, snake_case:

```html
<!-- ✅ так — унікальний клас на body -->
<body class="blog_page">
<body class="contacts_page">
<body class="product_page">
```

**Будова таблиці стилів:**

Усі селектори файлу секції / сторінки вкладаються в кореневий клас-сторінки:

```css
/* ✅ так */
.blog_page .hero { ... }
.blog_page .hero__title { ... }
.blog_page .shortcard { ... }
.blog_page .shortcard__thumb { ... }

/* ❌ ні — без кореневого scope */
.hero { ... }
.shortcard { ... }
```

У препроцесорі (SCSS/SASS) оформлюємо через вкладення:

```scss
// ✅ так
.blog_page {
  .hero { ... }
  .hero__title { ... }
  .shortcard { ... }
}
```

**Що залишається без page scope:**
- Глобальні блоки (`header`, `footer`, `popup`) — вони спільні й знаходяться в `global/`.
- Компоненти (`_btn.css`, `_form_field.css`) — теж глобальні, у `components/`.
- Партіали (`_breadcrumbs.css`, `_shortcard.css`) — базові стилі партіала глобальні; **переозначення зовнішнього вигляду під конкретну сторінку** — лише через page scope.

**Файлова структура:**
Файли секцій, прив'язаних до сторінки, лежать у `sections/<page>/` та імпортуються у відповідному page-блоку або з точки входу:

```
sections/
  blog/
    _blog_hero.css      ← стилі всередині .blog_page { ... }
    _blog_posts.css
  contacts/
    _contacts_hero.css
```

### 3.13 Гігієна

- Без закоментованого коду в комітах. Якщо потрібно тимчасово вимкнути — окремий PR, окрема гілка.
- Без невикористовуваних селекторів.
- Без дублювання правил — виносьте в загальний клас або utility.
- Без `!important`, крім явних утиліт (`.hidden`).
- Один селектор на рядок (`a, button { ... }` — окей; десять селекторів через кому в одному рядку — ні).
- Кодування — UTF-8, переклади рядків — LF.

## 4. Патерни компонентів

Розділ для повторюваних рішень, які мають вестись однаково по всьому проєкту.

### 4.1 Хлібні крихти (breadcrumbs)

**Структура:** `div.breadcrumbs > ul > li > a` — суворо, без винятків (включно з поточною сторінкою — теж `<a>`).

Візуально завжди **в один рядок** з **горизонтальним скролом**, без перенесення на наступний рядок. На мобільному це критично — інакше довгі шляхи ламають макет.

```html
<div class="breadcrumbs">
  <ul>
    <li><a href="#">Home</a></li>
    <li><a href="#">Catalog</a></li>
    <li><a href="#">Product name</a></li>
  </ul>
</div>
```

```css
.breadcrumbs ul {
  display: flex;
  align-items: center;
  overflow-x: auto;
  white-space: nowrap;
}
.breadcrumbs li { flex-shrink: 0; }

/* прихований скролбар */
.breadcrumbs ul::-webkit-scrollbar { width: 0; height: 0; }
```

Ключове:

- Розмітка завжди `div.breadcrumbs > ul > li > a` — однакова в усьому проєкті.
- `display: flex` + `flex-shrink: 0` на `<li>` — щоб елементи не стискалися.
- `overflow-x: auto` + `white-space: nowrap` — горизонтальний скрол замість wrap.
- Скролбар на breadcrumbs **ховаємо** (через `::-webkit-scrollbar { height: 0 }`), бо візуально він зайвий.

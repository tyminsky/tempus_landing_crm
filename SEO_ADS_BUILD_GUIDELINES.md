# Вказівки для побудови на основі `seo-ads.html`

Цей документ описує правила розширення верстки, побудованої на базі `seo-ads.html`.
Мета — додавати новий функціонал і стилі **без перевизначення** наявних стилів та скриптів.

---

## Базові принципи

1. **Використовуй наявні елементи.** Для побудови бери елементи, які вже є в `seo-ads.html`. Не вигадуй нову структуру там, де є готова.
2. **Наявні стилі та скрипти можна використовувати, але не перевизначати.** Наявні стилі/скрипти лежать усередині `seo-ads.html` — їх правити не можна, лише доповнювати.
3. **Новий CSS — в окремому `.css` файлі.** Не додавай стилі inline і не чіпай наявні `<style>`.
4. **Новий JS — в окремому `.js` файлі.** Не редагуй наявні `<script>`.

---

## Правило ізоляції стилів (клас-обгортка)

Щоб нові стилі не перевизначали наявні, **новий CSS-клас додається як модифікатор на блок-обгортку** `<div class="audit">`.

Було:

```html
<div class="audit">
  ...
</div>
```

Стало (додано унікальний клас-обгортку):

```html
<div class="audit audit_landing_2025">
  ...
</div>
```

> **Важливо:** клас-обгортка має бути **унікальним**. Не використовуй загальні чи плейсхолдерні назви (`new_class`, `wrapper`, `block` тощо) — обери власну змістовну й неповторну назву під конкретний блок/сторінку, напр.: `audit_landing_2025`, `audit_promo_offer`, `audit_dark_compact`. Унікальність гарантує, що нові правила не зачеплять інші блоки.

Усі нові селектори в `.css` файлі **обов'язково** прив'язуються до цього унікального класу:

```css
/* Правильно — стилі діють лише всередині унікальної обгортки */
.audit_landing_2025 .some-element {
  ...
}

/* Заборонено — перевизначає наявний стиль глобально */
.some-element {
  ...
}
```

Такий підхід дає block-scoped ізоляцію: стилі не «протікають» на інші блоки й не конфліктують із базовими правилами `seo-ads.html`.

---

## Заборонено

- Редагувати наявні стилі (`<style>`) усередині `seo-ads.html`.
- Редагувати наявні скрипти (`<script>`) усередині `seo-ads.html`.
- Писати селектори без прив'язки до унікального класу-обгортки.
- Перевизначати наявні класи (`.audit`, базові компоненти тощо) напряму.
- Inline-стилі (`style="..."`) та inline-скрипти замість окремих файлів.
- Використовувати неунікальні/плейсхолдерні назви класу-обгортки.

---

## Дозволено

- Підключати й використовувати наявні класи, компоненти та функції з `seo-ads.html`.
- Додавати новий унікальний клас-модифікатор на обгортку.
- Створювати нові `.css` та `.js` файли з власним функціоналом.
- Розширювати поведінку через нові скрипти, що не змінюють наявні.

---

## Структура файлів

```
seo-ads.html        ← базова верстка + наявні стилі/скрипти (наявне не редагувати)
{unique}.css        ← нові стилі (всі селектори під .{unique})
{unique}.js         ← нові скрипти
```

де `{unique}` — твоя унікальна назва класу-обгортки.

Підключення нових файлів (у `<head>` / перед `</body>`):

```html
<link rel="stylesheet" href="{unique}.css">
<script src="{unique}.js" defer></script>
```

---

## Чек-лист перед здачею

- [ ] Усі елементи взяті з наявних у `seo-ads.html`.
- [ ] Наявні стилі/скрипти всередині `seo-ads.html` не змінювалися.
- [ ] На блок додано **унікальний** клас-обгортку: `<div class="audit {unique}">`.
- [ ] Назва класу не плейсхолдерна й не загальна.
- [ ] Кожен новий CSS-селектор починається з `.{unique}`.
- [ ] Новий CSS — в окремому `.css` файлі.
- [ ] Новий JS — в окремому `.js` файлі.
- [ ] Немає inline-стилів та inline-скриптів.
- [ ] Немає перевизначення наявних класів.

## 1. Загальна логіка кластеризації {#логіка}

Запити розбито за **стадією воронки**:

| Стадія | Тип інтенту | Готовність | Рекомендація |
|---|---|---|---|
| Усвідомлення проблеми | Інформаційний | Холодна | Pmax / ремаркетинг |
| Порівняння рішень | Комерційний-дослідницький | Тепла | Пошук (ставки нижче) |
| Готовність до дії | Комерційний-транзакційний | Гаряча | Пошук (головний фокус) |

### Принципи message match для RSA-оголошень

1. **Заголовок 1 (30 симв):** містить ключове слово з запиту або синонім.
2. **Заголовок 2 (30 симв):** конкретна вигода / диференціатор («від 1 дня», «GCLID + UTM», «OpenCart / Horoshop»).
3. **Заголовок 3 (30 симв):** CTA або соц-доказ («Забронювати дзвінок», «Безкоштовний аудит»).
4. **Описи:** розкривають біль → рішення → відмінність від CRM-вендора.

## Пошук vs Pmax — розподіл кластерів {#пошук-vs-pmax}

### Для Пошукової кампанії (головний пріоритет)

| Кластер | Пріоритет | Обґрунтування |
|---|---|---|
| К2 — Інтеграція / налаштування CRM | ⭐⭐⭐ Найвищий | Транзакційний інтент, шукають виконавця, менше прямих конкурентів-інтеграторів |
| К3 — Наскрізна аналітика / конверсії | ⭐⭐⭐ Найвищий | Головна больова точка під рекламний сегмент (е-com + Google Ads) |
| К6 — Міграція з 1С / Bitrix | ⭐⭐ Високий | Змушені мігрувати = висока мотивація купити зараз |
| К4 — Телефонія + кол-трекінг (вузькі) | ⭐⭐ Середній | Тільки запити «телефонія crm», «кол-трекінг google ads» — не широкі телефонні |
| К1 — Вибір CRM (вузькі) | ⭐ Обережно | Конкурентно, дорого, але кілька запитів типу «підбір crm для магазину» можуть зайти |

### Для Pmax / ремаркетингу

| Кластер | Чому Pmax |
|---|---|
| К1 — Вибір CRM (широкі) | Дорогий аукціон, краще банером/відео нагадати після першого контакту |
| К5 — Омніканальність | Менш частотний пошук, але аудиторія добре реагує на банери з болем («хаос у переписках») |
| К3 — Аналітика (холодний) | Широкі запити «що таке наскрізна аналітика» — інформаційний, краще утеплювати через Pmax |
| Ремаркетинг | Всі, хто побував на лендінгу — повернути через Pmax з варіантом B або C оферу |

## Додаток: пріоритетні RSA-оголошення для першого тесту

### Група «Інтеграція CRM» (К2)

**RSA #1**
- H1: Інтеграція CRM з магазином
- H2: OpenCart / WooCommerce / Horoshop
- H3: Заявки в CRM від 1 дня
- Опис: Підключаємо KeyCRM, KeepinCRM до вашого сайту. Передаємо UTM, GCLID, замовлення. З вас — лише доступи.

**RSA #2**
- H1: Налаштування KeyCRM / KeepinCRM
- H2: Інтегратор — не продавець підписки
- H3: Забронювати дзвінок
- Опис: Впроваджуємо CRM під ваш бізнес: воронки, телефонія, аналітика. Говоримо з інженером, а не менеджером.

### Група «Аналітика + Google Ads» (К3)

**RSA #1**
- H1: Реклама зливає бюджет?
- H2: Підключіть CRM до Google Ads
- H3: Безкоштовний аудит
- Опис: Налаштуємо передачу GCLID, офлайн-конверсій та серверний GTM. Побачите, які кампанії реально продають.

**RSA #2**
- H1: Серверний GTM + CRM інтеграція
- H2: GCLID / JBRAID / офлайн-конверсії
- H3: Дізнатись вартість
- Опис: Правильна аналітика для інтернет-магазину — раз зробили, працює роками. Не як у 80% спеціалістів.

### Група «Міграція 1С / Bitrix» (К6)

**RSA #1**
- H1: Мігруєте з 1С або Bitrix24?
- H2: Перенесемо без втрат даних
- H3: Підібрати CRM → дзвінок
- Опис: Склад, клієнти, замовлення — переносимо в KeyCRM, KeepinCRM або SalesDrive. За кілька днів, нічого не зламаємо.
